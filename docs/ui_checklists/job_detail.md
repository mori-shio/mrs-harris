# UI Checklist: ジョブ詳細画面

このファイルはジョブ詳細画面 (`/jobs/:name` など) における表示レベル（UI）の要件を管理するチェックリストです。修正を行った際は、対象の要件を満たしているかを検証してください。

## ジョブ概要・基本情報
- [x] **最新バージョンの表示**:
  ジョブ概要の更新日時の右側に「Key: 最新バージョン Value: vXX」の形式で表示されること。文字色は他の項目と同じく白色（テーマ標準色）であること。
- [x] **最新バージョンの整合性**:
  表示されるバージョン番号が常に実際の最新のバージョン番号（例: `v3` 等）と一致していること。`v0` などの初期値に固定されないこと。
- [x] **ジョブIDの非表示**:
  URLパスがジョブ名ベースになったため、ジョブの概要カード内にジョブID（内部的なID）を表示しないこと。
- [x] **ジョブ実行通知の toast 化**:
  右上の「実行」を押した際、ブラウザ標準ダイアログではなく、画面内 toast でジョブ実行トリガー完了が通知されること。


## タブ・詳細表示
 - **シェルコマンドのアンラップ表示**:
  「シェルコマンド」タブにおいて、実行スクリプトの表示から `sh -c` やラッパーの記述が取り除かれ、ユーザーが実際に入力したスクリプト本体のみが正しく表示されること。
- [x] **実行履歴テーブルのソートヘッダー**:
  「実行履歴」タブ内の表で、実行カラムのヘッダーがプルダウン（`<select>`）ではなく、ヘッダー自体をクリックして昇順/降順をトグルできる自然なUIになっていること。
- [x] **実行番号の永続カラム表示**:
  「実行履歴」タブ内の実行番号は `job_runs.run_number` の値を直接表示し、表示時に `COUNT(*)` で再計算されないこと。
- [x] **実行履歴ステータス表記の統一**:
  実行履歴タブ内のステータスが、必要な状態ごとに統一された文言とバッジ見た目で表示され、同じ状態で表記揺れが起きないこと。
- [x] **終端状態の実行時間表示**:
  実行履歴タブでは、`running` 以外の終端状態（`succeeded / failed / cancelled / dead_letter`）であれば、実行時間が `-` ではなく表示されること。
- [x] **実行履歴ポーリング表示の非表示**:
  実行履歴カード右上に `5秒で更新` の表示を出さず、見た目を崩さないこと。ポーリング自体は裏側で継続すること。
- [x] **実行履歴タブ表示中のみ常時ポーリング**:
  ジョブ詳細画面では、他ユーザーの実行も拾えるよう `実行履歴` タブを開いている間は常に 5 秒 polling し、別タブへ移動したら polling を止めること。
- [x] **アーカイブ済みログ状態の表示**:
  実行履歴タブでは、終了済み run のログが `job_logs` ではなくアーカイブから再表示される状態になっている場合、その run がアーカイブ済みであることを補助的に判別できる表示があること。
- [x] **直接設定環境変数の textarea 表示**:
  環境変数タブの「直接設定された環境変数」は表形式ではなく、ジョブ作成/編集画面と同じ `KEY=VALUE` の複数行テキストを readonly textarea でそのまま表示すること。

## バージョン履歴・AB比較
- [x] **新規ジョブの初期履歴作成**:
  新しく作成したジョブを詳細画面で開いた時点で、設定変更履歴タブに少なくとも `v1` が表示されること。
- [x] **AB比較ボタンの動作**:
  バージョンの表で「A」と「B」のラジオボタンを正しく選択して「ABでの差分比較」ボタンを押した際、「比較する2つのバージョン（AとB）を選択してください。」という不要なエラーアラートが表示されずに、比較処理が実行される（または比較モーダルが開く）こと。
- [x] **AB比較未選択エラーの画面内 notice 化**:
  A または B が未選択の状態で「ABでの差分比較」を押した際、ブラウザ標準のアラートではなく、履歴カード内に inline notice が表示されること。
- [x] **ジョブ削除確認の画面内ダイアログ化**:
  右上の「削除」を押した際、ブラウザ標準の確認ダイアログではなく、アプリ内の確認ダイアログが表示され、確認後に削除処理が進むこと。

## モーダル全般
 - **背景スクロールのロック**:
  AB比較やバージョンの詳細などのモーダルが表示されている間、モーダル内をスクロールした際に背後の画面が連動してスクロールされない（スクロールロックが効いている）こと。
- [x] **モーダル表示時の横ずれ防止**:
  ジョブ詳細画面で削除確認モーダルや各種モーダルを開いた際、右側スクロールバーの消失によって背景レイアウトが横にずれたりちらついたりしないこと。
 - ジョブの概要カード内に「ジョブID」が表示されていないこと（URLでジョブ名がパスになるため不要）
 - ジョブの概要カード内の「最新バージョン」が、正しく `v1`, `v2` などのフォーマットで実際の履歴からインクリメントされて表示されること（`v0`にならないこと）
 - 実行履歴テーブルのヘッダーにある「実行カラムの並び替えボタン」が、スペース画面の表の並び替えUIとデザイン上統一されていること
- [x] **新規ジョブの初回実行履歴表示**:
  新しく作成したジョブで「実行」を行ったあと、実行履歴タブに `#1` の run が表示されること。
