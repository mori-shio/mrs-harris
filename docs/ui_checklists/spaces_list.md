# UI Checklist: スペース一覧画面

このファイルはスペース一覧画面 (`/spaces`) における表示レベル（UI）の要件を管理するチェックリストです。修正を行った際は、対象の要件を満たしているかを検証してください。

## 削除確認
- [x] **スペース削除確認の画面内ダイアログ化**:
  一覧の各行で「削除」を押した際、ブラウザ標準の確認ダイアログではなく、アプリ内の確認ダイアログが表示され、確認後に削除処理が進むこと。
- [ ] **並べ替えUIの削除**:
  スペース一覧ではレコード並べ替え機能を廃止し、左端の6点リーダアイコンおよびドラッグ操作が表示されないこと。
- [ ] **スペース名リンクの遷移先統一**:
  スペース一覧のスペース名リンクを押すと、スペース詳細画面ではなく `/jobs?space={space_id}` に遷移すること。
- [ ] **スペース名リンクの遷移補助表示**:
  スペース一覧のスペース名リンクには別画面遷移を示す `□↗` 相当のアイコンを表示し、行 hover 時はスペース名だけ青色、スペース名 hover 時は青色 + 下線を適用すること。
- [ ] **ジョブ数カラムの左寄せ**:
  スペース一覧の `ジョブ数` カラムでは、数字バッジが左寄せで表示されること。
- [ ] **優先度カラムの表示**:
  スペース一覧の `作成日時` 表示は廃止し、代わりに `優先度` カラムを表示すること。
- [ ] **優先度順での整列**:
  スペース一覧のレコードは `優先度` の小さい順に並ぶこと。
- [ ] **操作カラムヘッダの非表示**:
  スペース一覧の `操作` カラムでは、ヘッダ行にラベル文字を表示しないこと。
- [ ] **新規スペース作成のモーダル化**:
  スペース一覧画面の `新規スペース作成` を押すと、ジョブ一覧画面の `+` ボタンと同じ内容のモーダルが開くこと。
- [ ] **スペース編集のモーダル化**:
  スペース一覧画面の `編集` を押すと、専用編集画面ではなく同じモーダルが編集モードで開き、現在値が初期表示されること。
- [ ] **モーダルフォームの明示 submit 化**:
  スペース一覧画面の新規/編集モーダルでは、`Enter` キーによる暗黙 submit を行わず、保存ボタンの明示クリック時だけ処理が実行されること（`textarea` の改行は許可すること）。
- [ ] **スペースモーダルの優先度入力**:
  スペース一覧の新規/編集モーダルでは、`優先度` の整数入力欄を表示し、保存時にその値が反映されること。
