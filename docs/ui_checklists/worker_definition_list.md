# UI Checklist: ワーカー定義一覧画面

このファイルはワーカー定義一覧画面 (`/worker-definitions`) における表示レベル（UI）の要件を管理するチェックリストです。修正を行った際は、対象の要件を満たしているかを検証してください。

## 表示内容
- [ ] **タイトル表記の簡素化**:
  一覧画面のタイトルは `ワーカー定義` と表示されること。
- [ ] **ワーカー名左アイコンの非表示**:
  ワーカー名リンクの左側には補助アイコンを表示しないこと。
- [ ] **ワーカー名リンク hover の共通化**:
  表のレコード hover 時にはワーカー名リンクが青色になり、ワーカー名リンク自身への hover 時には青色 + 下線になること。
- [ ] **ワーカー名リンクの当たり判定最小化**:
  ワーカー名リンクは文字幅の範囲だけがクリック/ホバー対象になり、文字がない右側の空白には当たり判定が残らないこと。
- [ ] **名前ベースURL**:
  一覧画面から詳細・編集へ遷移するワーカー定義リンクは `id` ではなくワーカー定義名ベースの URL になっていること。
- [ ] **操作カラムヘッダの非表示**:
  操作カラムのヘッダ文字 `操作` は表示しないこと。
- [ ] **操作カラムの並び替え UI 非表示**:
  操作カラムには並び替えボタンや並び替え可能に見える UI を表示しないこと。
- [ ] **削除後 toast の表示**:
  ワーカー定義詳細画面から削除したあと、一覧へ戻った際に右上 toast で `ワーカー定義を削除しました。` が表示されること。
- [x] **新規作成ボタン表記の統一**:
  ワーカー定義一覧画面の右上作成ボタンは `+ 新規作成` と表示されること。
