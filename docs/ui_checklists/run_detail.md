# UI Checklist: 実行詳細画面

このファイルはジョブの実行詳細画面 (`/jobs/:name/runs/:run_id` など) における表示レベル（UI）の要件を管理するチェックリストです。

## 実行ログ
 - **実行ログの表示**:
  詳細画面を開いた際、実行ログ（stdout/stderr 等）が正しく読み込まれ、空にならずに表示されること。（ローカルエコーのジョブなど、実行完了しているもの）
 - **実行ログカードの存在**:
  画面の読み込み中・完了後や、画面をリロードした際にも、実行ログを表示するためのカード要素自体が消えたり非表示になったりせず、常に存在していること。
 - [x] **Controller 実行ログの重複防止**:
  Controller ワーカーで逐次ログ表示を有効にした run では、同じ stdout/stderr 行が完了 callback によって二重保存・二重表示されないこと。実行中の live 表示でも、リロード後の再表示でも 1 回ずつだけ見えること。
 - [x] **実行ログ時刻のミリ秒表示**:
  実行詳細画面のログ時刻は `HH:MM:SS.mmm` 形式でミリ秒単位まで表示されること。
 - [x] **非 ASCII を含む xtrace 行の保持**:
  `set -x` で日本語など非 ASCII を含むコマンドを実行した場合でも、stderr 側 xtrace 行が欠落せず、必要に応じて shell 依存のエスケープ表現になっても後続の `+ sleep ...` などを含めて順に表示されること。
 - [x] **xtrace 行の非エラー色表示**:
  `set -x` による `+ ...` 行は stderr 由来でもエラーメッセージと同じ赤では表示せず、通常ログに近い中立的な色で表示すること。
 - [x] **xtrace 行の可読表示**:
  `set -x` の xtrace 行は、shell 依存の `$'\\343...'` のようなエスケープ表現がそのまま露出せず、可能な範囲で人が読める文字列に正規化して表示すること。
 - [x] **xtrace と stdout の表示順補正**:
  `echo` 系の xtrace 行とその stdout 行は、タイミングが近い場合でも `stdout -> xtrace` の逆転表示にならず、`xtrace -> stdout` の順で自然に読めること。
 - [x] **アーカイブ済み実行ログの表示**:
  実行終了後に `job_logs` から削除された run でも、アーカイブ済みであれば実行詳細画面のログが空にならず再表示できること。

## ステータス表示
 - [x] **実行ステータス表記の統一**:
  実行詳細画面のステータス表示が、ジョブ詳細の実行履歴と同じ文言・同じバッジルールで表示されること。
- [x] **終端状態の終了日時・実行時間表示**:
  実行詳細画面では、`running` 以外の終端状態（`succeeded / failed / cancelled / dead_letter`）であれば、終了日時と実行時間が `-` ではなく表示されること。
- [x] **終端状態での live polling 停止**:
  実行詳細画面では、`succeeded / failed / cancelled / dead_letter` など終端状態の run に対して `/runs/:id/live` への継続 polling を行わないこと。非終端状態の run では従来どおり live 更新が継続すること。

## 成功通知
 - [x] **実行キャンセル通知の toast 化**:
  「強制停止」を押した際、ブラウザ標準のアラートではなく、画面内 toast でキャンセル要求送信が通知されること。
