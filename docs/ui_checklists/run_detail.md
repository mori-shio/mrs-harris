# UI Checklist: 実行詳細画面

このファイルはジョブの実行詳細画面 (`/jobs/:name/runs/:run_id` など) における表示レベル（UI）の要件を管理するチェックリストです。

## 実行ログ
 - **実行ログの表示**:
  詳細画面を開いた際、実行ログ（stdout/stderr 等）が正しく読み込まれ、空にならずに表示されること。（ローカルエコーのジョブなど、実行完了しているもの）
 - **実行ログカードの存在**:
  画面の読み込み中・完了後や、画面をリロードした際にも、実行ログを表示するためのカード要素自体が消えたり非表示になったりせず、常に存在していること。
 - [x] **Controller 実行ログの重複防止**:
  Controller ワーカーで逐次ログ表示を有効にした run では、同じ stdout/stderr 行が完了 callback によって二重保存・二重表示されないこと。実行中の live 表示でも、リロード後の再表示でも 1 回ずつだけ見えること。
 - [x] **実行ログ時刻のミリ秒表示**:
  実行詳細画面のログ時刻は `HH:MM:SS.mmm` 形式でミリ秒単位まで表示されること。
 - [x] **非 ASCII を含む xtrace 行の保持**:
  `set -x` で日本語など非 ASCII を含むコマンドを実行した場合でも、stderr 側 xtrace 行が欠落せず、必要に応じて shell 依存のエスケープ表現になっても後続の `+ sleep ...` などを含めて順に表示されること。
 - [x] **xtrace 行の非エラー色表示**:
  `set -x` による `+ ...` 行は stderr 由来でもエラーメッセージと同じ赤では表示せず、通常ログに近い中立的な色で表示すること。
 - [x] **xtrace 行の可読表示**:
  `set -x` の xtrace 行は、shell 依存の `$'\\343...'` のようなエスケープ表現がそのまま露出せず、可能な範囲で人が読める文字列に正規化して表示すること。
 - [x] **xtrace と stdout の表示順補正**:
  `echo` 系の xtrace 行とその stdout 行は、タイミングが近い場合でも `stdout -> xtrace` の逆転表示にならず、`xtrace -> stdout` の順で自然に読めること。
- [x] **アーカイブ済み実行ログの表示**:
  実行終了後に `job_logs` から削除された run でも、アーカイブ済みであれば実行詳細画面のログが空にならず再表示できること。
- [x] **ログ保存状態の表示**:
  実行詳細画面では、実行ログが live な `job_logs` 由来なのか、終了後のアーカイブ由来なのかを補助的に判別できる表示があること。アーカイブ失敗時も、失敗状態が判別できること。
- [x] **ログカード右上の表示簡素化**:
  実行ログカード右上には接続状態のみを表示し、`アーカイブ済み (ローカル)` などのログ保存状態バッジは表示しないこと。
- [x] **接続状態の自動更新**:
  実行詳細画面を開いたまま run が完了した場合、実行ログカード右上の状態表示が `準備中` のまま残らず、自動で `接続切断 (完了)` へ切り替わること。
- [x] **完了済み run の初回ログ取得方式**:
  完了済み run の詳細画面では、初回ログ表示のために WebSocket を開かず、HTTP の 1 回取得だけでログを表示すること。

## ステータス表示
 - [x] **実行ステータス表記の統一**:
  実行詳細画面のステータス表示が、ジョブ詳細の実行履歴と同じ文言・同じバッジルールで表示されること。
- [x] **ジョブバージョン表示とモーダル導線**:
  実行ステータスカード内では `バージョン vXX` ではなく `ジョブバージョン vXX` と表示し、`vXX` 押下時にジョブ詳細と同じ readonly バージョン詳細モーダルが開くこと。
- [x] **ジョブ種別表記の英字統一**:
  実行詳細画面の `ジョブバージョン` モーダル内の `ジョブタイプ` は `Cron / OneShot` の英字表記のみを表示し、日本語補足を付けないこと。
- [x] **ジョブバージョンモーダルのレイヤー順**:
  実行詳細画面で `ジョブバージョン vXX` からモーダルを開いたとき、左側サイドバーより前面にモーダルと backdrop が表示されること。
- [ ] **ジョブバージョンモーダルのヘッダー遮蔽**:
  実行詳細画面で `ジョブバージョン vXX` からモーダルを開いたとき、固定ヘッダー部分まで backdrop/シャドウで覆われ、ジョブ詳細画面のモーダルと同じ見え方になること。
- [ ] **共通バージョンモーダルの余白調整**:
  共通バージョンモーダルは左右幅が少し抑えられ、画面上下端とのマージンが現状より広く取られていること。
- [x] **ジョブバージョンモーダル右端余白**:
  ジョブバージョンモーダル内の各カード右端とモーダル本文右端のあいだに十分な余白があり、スクロールバー表示時も右側だけ詰まって見えないこと。
- [x] **終端状態の終了日時・実行時間表示**:
  実行詳細画面では、`running` 以外の終端状態（`succeeded / failed / cancelled / dead_letter`）であれば、終了日時と実行時間が `-` ではなく表示されること。
- [x] **終端後の live polling 終了**:
  実行詳細画面では、run が終端状態へ遷移した直後も archive 状態の反映に必要な間は live 更新を継続し、`アーカイブ済み` または `アーカイブ失敗` など最終表示へ到達したら polling が止まること。
- [x] **実行完了直後の自動画面更新**:
  実行詳細画面を開いたまま run が終端状態へ遷移した場合、画面をリロードしなくても右上の「強制停止」ボタンが消え、ログ保存の archive badge も最新状態へ自動更新されること。
- [x] **実行 ID の非表示**:
  実行詳細画面の実行ステータスサマリには内部向けの実行 ID を表示せず、そのスペースに `ログ保存` を無理なく収められること。

## 成功通知
 - [x] **実行キャンセル通知の toast 化**:
  「強制停止」を押した際、ブラウザ標準のアラートではなく、画面内 toast でキャンセル要求送信が通知されること。
