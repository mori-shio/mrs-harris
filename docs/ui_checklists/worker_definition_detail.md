# UI Checklist: ワーカー定義詳細画面

このファイルはワーカー定義詳細画面 (`/worker-definitions/:id`) における表示レベル（UI）の要件を管理するチェックリストです。修正を行った際は、対象の要件を満たしているかを検証してください。

## 削除確認
- [x] **ワーカー定義削除確認の画面内ダイアログ化**:
  右上の「削除」を押した際、ブラウザ標準の確認ダイアログではなく、アプリ内の確認ダイアログが表示され、確認後に削除処理が進むこと。

## 表示内容
- [x] **Controller ワーカー定義の排除**:
  `controller` タイプのワーカー定義が migration 後に詳細画面へ残らず、既存のローカル検証用定義は `lambda` または `fargate` として表示されること。
- [ ] **タイトルから接頭辞を除去**:
  詳細画面のタイトルは `ワーカー定義: <name>` ではなく、ワーカー名そのものだけを表示すること。
- [ ] **ジョブ詳細と同系統のレイアウト**:
  パンくず、ヘッダ、概要カード、下位カードの構成がジョブ詳細画面と同系統の密度と配置になること。
- [ ] **ステータス表示の削除**:
  ワーカー定義詳細画面では `有効/無効` のステータス表示を出さないこと。
- [ ] **下位カードのタブ化**:
  概要カードの下は、ジョブ詳細画面のようにタブで `接続構成` と `設定変更履歴` を切り替えられること。
- [ ] **タブURLの同期**:
  ワーカー定義詳細画面のタブ切り替え時は `?tab=config` / `?tab=history` が URL に反映されること。
- [ ] **タブ初期表示のフォールバック**:
  `?tab` が無いときは先頭タブ（`接続構成`）が開き、未知の値でも先頭タブへフォールバックすること。
- [ ] **設定変更履歴タブの初期化**:
  `/worker-definitions/:id` を通常遷移・HTMX 由来の遷移どちらで開いても、`設定変更履歴` タブを押下すると確実に内容へ切り替わること。
- [ ] **設定変更履歴の表示**:
  ワーカー定義詳細画面では、バージョン番号・変更ユーザ・変更日時を含む設定変更履歴が確認できること。
- [ ] **履歴テーブルの静的ヘッダ表示**:
  設定変更履歴テーブルの各カラムヘッダには、並び替えボタンや並び替えアイコンを表示しないこと。
- [ ] **履歴バージョンの readonly モーダル表示**:
  設定変更履歴の `vX` を押すと、ジョブ詳細のバージョンモーダルと同系統の readonly モーダルが開き、その時点のワーカー定義設定を確認できること。
- [x] **概要カードに最新バージョンを表示**:
  概要カードの中に最新のバージョン番号（vX）が表示されること。
- [x] **下位カードのタイトルデザイン統一**:
  接続構成・設定変更履歴のカードタイトルがジョブ詳細画面のタブ下のカードと同様のデザインになっていること。
- [x] **バージョンモーダルのデザイン統一**:
  バージョン詳細を開いた際、モーダルのコンテナがジョブ詳細画面の差分比較モーダルと同様の左メニューまで覆う全画面シャドウ・アニメーション・ヘッダーデザインとなっていること。
- [x] **AB差分比較機能のサポート**:
  設定変更履歴テーブルに比較用ラジオボタンを表示し、「ABでの差分比較」からジョブ詳細画面と同様に2つのバージョンの差分を確認できること。
- [x] **AB差分比較の未選択案内**:
  A または B が未選択の状態で「ABでの差分比較」を押した際、ブラウザ標準のアラートではなく、1 ボタンのアプリ内モーダルで案内されること。
- [x] **AB差分比較モーダルのオーバーレイ表示**:
  `ABでの差分比較` を押したとき、差分比較の内容はページ下部にインライン表示されず、画面中央のモーダルオーバーレイとして表示されること。
- [x] **AB差分比較の遷移方式非依存**:
  通常遷移・`hx-boost` 遷移のどちらで詳細画面へ入っても、設定変更履歴タブの「ABでの差分比較」ボタンが動作すること。
- [x] **履歴が 1 件のときの比較ボタン無効化**:
  設定変更履歴が 1 件しかない場合、`ABでの差分比較` ボタンは無効化され、差分比較には 2 つ以上のバージョンが必要であることが明示されること。
- [x] **各種バージョンモーダルの共通デザイン化**:
  バージョンモーダルおよび差分比較モーダルが共通の CSS クラス（`detail-modal-overlay` 等）とコンポーネントを用いて描画されていること。
