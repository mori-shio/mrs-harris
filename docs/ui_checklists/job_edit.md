# UI Checklist: ジョブ編集画面

このファイルはジョブ編集画面 (`/jobs/new` および `/jobs/:name/edit`) における表示レベル（UI）の要件を管理するチェックリストです。修正を行った際は、対象の要件を満たしているかを検証してください。

## ヘッダー・ナビゲーション
 - **パンくずリスト**:
  既存のジョブを編集する際、パンくずリストの構成が「ホーム / ジョブ / {ジョブ名} / 編集」となっていること。
  新規作成の場合は「ホーム / ジョブ / ジョブ 新規作成」となっていること。
- [x] **パンくずリンクの hover 表示**:
  パンくずリスト内のリンク要素にホバーした際、リンク文字が青色になり、下線が表示されること。
- [x] **タイトル横の戻るボタン hover 表示**:
  画面タイトル左側の戻るボタン（`←` アイコン）にホバーした際、アイコン色が青色に変わること。
- [x] **タイトル横の戻るボタンの当たり判定**:
  画面タイトル左側の戻るボタン（`←` アイコン）は、見た目を大きく崩さずに少し広めの hover / click 領域を持つこと。

## フォーム入力
- [x] **ジョブ種別選択肢の英字統一**:
  ジョブ作成/編集画面の `ジョブタイプ` 選択肢は `Cron / DAG / OneShot` の英字表記のみを表示し、日本語補足を付けないこと。
 - **ジョブ名の重複バリデーション**:
  ジョブ名入力のテキストボックスからカーソルが外れたとき（`blur` イベント時など）に、ジョブ名が既存のものと重複している場合、テキストボックスの下に赤文字で重複している旨のエラーメッセージが表示されること。
 - パンくずリストが「ホーム / ジョブ / {job name} / 編集」の形式になっていること
 - ジョブ名が他のジョブと被っていた場合、保存を押した時にバックエンドでエラーにならず正しくエラー処理（または保存処理）がなされること
 - DAGジョブやリトライ設定が無効なジョブの編集時に「保存」を押した際、正しくフォームが送信され、リダイレクトが機能すること（非表示フィールドの必須やminバリデーションエラーによる無反応が起きないこと）
 - ジョブ編集・保存後、新しいジョブ名（変更した場合）のURLへ正しくリダイレクトされること
- [x] **Controller ワーカー定義の非表示**:
  実行ワーカーの選択肢に `controller` タイプのワーカー定義が表示されず、既存のローカル検証用定義も `Fargate` または `Lambda` として表示されること。
- [ ] **コピーして作成の初期値反映**:
  `/jobs/new?copy_from=<job_name>` で開いた新規作成画面では、ジョブ名だけ空欄で、説明・ジョブタイプ・ワーカー・スペース・スケジュール・スクリプト/DAG・環境変数・SSM・タイムアウト・リトライ・タグ・Slack通知・有効状態がコピー元から初期値として反映されること。
