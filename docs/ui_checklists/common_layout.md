# UI Checklist: 共通レイアウト

このファイルは共通レイアウト（ヘッダー、サイドバー、メインコンテンツ領域）における表示レベル（UI）の要件を管理するチェックリストです。修正を行った際は、対象の要件を満たしているかを検証してください。

- [ ] **ブランド表示のヘッダー統合**:
  左側メニューペイン上部の `Mrs. Harris` 表示は独立ブロックとして残さず、画面全体のヘッダー内へ小さめの表示として統合されること。
- [ ] **サイドバー開閉トグル**:
  ヘッダー左側のトグル操作でメニューペインを隠したり再表示したりできること。
- [ ] **サイドバー折りたたみ時のレイアウト追従**:
  サイドバーを隠した時にメインコンテンツ領域が自然に広がり、重なりや不自然な余白が出ないこと。
- [ ] **将来の右側ヘッダー領域の確保**:
  ヘッダー右側に将来的なログインユーザ情報表示のための領域が残されていること。
- [ ] **パンくず hover 挙動の全画面統一**:
  パンくず内のリンクは、ジョブ一覧を含む全画面で hover 時に同じ青色 + 下線の見た目になること。
- [x] **パンくず UI の共通化**:
  パンくずリストは各画面で個別HTMLを直書きせず、共通 macro と共通 `BreadcrumbItem` データで描画されること。
- [x] **パンくず主要項目のアイコン表示**:
  パンくずリストの `ジョブ`、`ステップフロー`、`スペース`、`ワーカー定義` などの主要項目では、サイドバーと同じアイコンが左側に表示されること。
- [ ] **ログイン画面の中央配置**:
  ログイン画面では、タイトルと「ようこそ」カードが画面中央に配置されること。
