# UI Checklist: ワーカー定義作成・編集画面

このファイルはワーカー定義作成・編集画面 (`/worker-definitions/new`, `/worker-definitions/:id/edit`) における表示レベル（UI）の要件を管理するチェックリストです。修正を行った際は、対象の要件を満たしているかを検証してください。

## バックエンド種別
- [x] **新規作成画面タイトル**:
  ワーカー定義の新規作成画面のタイトルは `ワーカー定義 新規作成` と表示されること。
- [x] **タイトル位置のジョブ作成画面揃え**:
  ワーカー定義作成/編集画面のタイトル行は、ジョブ新規作成画面と同じようにフォームカード幅に揃った位置に配置されること。
- [x] **有効化トグルの右上配置**:
  `ノード有効化` のトグルは独立した設定ブロックとしてカード内上部に置かず、ジョブ新規作成画面と同様にタイトル行の右上に配置されること。
- [x] **フォーム横幅のジョブ作成画面揃え**:
  ワーカー定義作成/編集画面のカード幅はジョブ新規作成画面と同じ幅に揃っていること。
- [x] **ヘッダー説明文の非表示**:
  タイトル直下の補足説明文は表示しないこと。
- [x] **UI文言のワーカー統一**:
  画面内のユーザー向け文言では `ノード` ではなく `ワーカー` を使うこと。
- [x] **基本情報カードへの集約**:
  ワーカー定義作成/編集画面では、`ワーカー名`、`説明`、`ワーカータイプ` が `基本情報` カード内にまとまって表示されること。
- [x] **ワーカータイプ別カードの出し分け**:
  `ワーカータイプ` の選択に応じて、次のカードが `ワーカータイプ: Fargate` または `ワーカータイプ: Lambda` として切り替わり、選択していない側のカードは表示されないこと。
- [x] **Controller ワーカー種別の除去**:
  ワーカーのバックエンド種別の選択肢に `controller` が表示されず、`fargate` と `lambda` だけを選択できること。
- [x] **ローカル検証導線の表示**:
  ローカル検証が必要な場合は `lambda.function_name = "local"` または `fargate.cluster_arn = "local"` を使う旨が、フォーム上の補足で分かること。
