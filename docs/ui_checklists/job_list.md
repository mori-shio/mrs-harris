# ジョブ一覧画面 UIチェックリスト

 - スペース選択タブについて、「すべて」や「未分類」も含めて、URLと合致する1つのタブだけがアクティブ状態（スタイル）になること
 - アクティブでないタブは通常の見た目であり、ホバーした時のみホバー用のスタイルになること
 - 「実行」を押した際、ブラウザ標準のアラートではなく、画面内 toast で実行トリガー完了が通知されること
 - ジョブ一覧の `タイプ` 列では `Cron / OneShot` の英字表記のみを表示し、`DAG` や日本語補足を出さないこと
 - ジョブ一覧の `スケジュール` 列では未設定時に `-` を表示し、`なし` や `未設定` の表記揺れを出さないこと
 - ジョブ一覧の `タグ` 列では、表示が複数行に折り返される場合に通常時と同じチップ見た目のまま先頭数件 + `⋯` へ省略し、その省略状態の行にのみホバー時ツールチップで全タグを表示すること
 - ジョブ一覧画面の `コピーして作成` ボタンを押すと、既存ジョブを前方一致検索できるモーダルが開くこと
 - `コピーして作成` モーダルで既存ジョブを選択すると `/jobs/new?copy_from=<job_name>` へ遷移すること
 - `コピーして作成` モーダルでは、入力した文字列と前方一致している既存ジョブ名の一致部分だけが青色で強調表示されること
 - `コピーして作成` モーダルは初期表示時のサイズを維持し、検索結果が少ない状態や空状態でもモーダル全体の高さが縮まらないこと
 - ジョブ一覧画面内のモーダルフォームでは、`Enter` キーによる暗黙 submit を行わず、保存系ボタンの明示クリック時だけ処理が実行されること（`textarea` の改行は許可すること）
 - ジョブ一覧画面のスペース新規作成モーダルでは、`優先度` の整数入力欄を表示し、その値を送信できること
 - ジョブ一覧画面のスペース選択タブは、`すべて` の次にスペースの `優先度` が小さい順で並び、値が大きいスペースほど右側に表示されること
 - ジョブ一覧画面のスペース選択タブは、横スクロール自体は可能なまま、スクロールバーは表示しないこと
 - ジョブ一覧画面のスペース選択タブは、左右にまだ隠れているタブがある時だけ、その方向の端にシャドウを表示すること
 - ジョブ一覧画面のスペース選択タブ端のシャドウは、隠れているタブの存在が一目で分かる程度に十分視認できること
 - ジョブ一覧画面のスペース選択タブ端のシャドウは、境界線が過度に目立たず、シャドウとして自然に見えること
 - ジョブ一覧画面のスペース選択タブでは、通常時の下線とホバー時の白線が同じ位置に揃い、上下にずれて見えないこと
 - ジョブ一覧画面のスペース選択タブでは、ホバー時の下線が通常時より十分視認できる濃さで表示されること
 - ジョブ一覧画面のスペース選択タブでは、active タブにホバーした場合も下線の強調が視認できること
 - ジョブ一覧画面のスペース選択タブでは、active / non-active を問わずホバー時の下線強調が明確に判別できること
 - ジョブ一覧画面のジョブ名リンクは、スペース一覧のスペース名リンクと同様に文字幅の範囲だけがクリック/ホバー対象になり、文字がない右側の空白には当たり判定が残らないこと
 - ジョブ一覧画面の右上作成ボタンは `+ 新規作成` と表示されること
 - ジョブ一覧画面でスペースタブを `すべて` 以外に切り替えた場合、パンくず末尾に選択中スペースが表示され、登録済みスペースは `folder`、未分類は `help-circle` アイコンが表示されること
 - ジョブ一覧画面では、スペースタブ下の検索カードを表示しないこと
 - ジョブ一覧画面のページ右上アクションでは、`コピーして作成` ボタンの左側にフリーワード検索窓を表示し、入力値でジョブ名/説明を前後左右部分一致検索できること
 - ジョブ一覧画面のフリーワード検索では、一覧上でヒットしたジョブ名の一致部分が青色で強調表示されること
